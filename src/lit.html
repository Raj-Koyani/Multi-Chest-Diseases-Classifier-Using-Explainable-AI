Proposed Architecture and Methodology

1. Data Collection and Preprocessing

Three public chest X-ray datasets were merged, resulting in ~43,000 images across five categories: COVID-19, Tuberculosis, Viral Pneumonia, Lung Opacity, and Normal.

Preprocessing included: resizing to 300–382 px, grayscale to RGB conversion, normalization using ImageNet mean/std, and augmentation (rotation ±15°, horizontal flip, brightness/contrast adjustments, CutMix/MixUp).

2. Model Architecture

The model architecture is based on DenseNet121 + CBAM:

  • Backbone: DenseNet121 pretrained on ImageNet for feature extraction.
  • Attention Module: Convolutional Block Attention Module (CBAM) to highlight radiologically relevant regions.
  • Classification Head: Global Average Pooling → Dense(512, ReLU) → Dropout(0.4) → Softmax (5 outputs).

3. Training Strategy

Input images were preprocessed at 300–382 px. Training was performed on GPU with weighted categorical cross-entropy to address class imbalance. Optimizer used was AdamW with cosine learning rate scheduler. Batch size was 32, with an 80/10/10 train-validation-test split and 5-fold cross-validation. The target was to achieve 95%+ accuracy and AUROC ≥ 0.97.

4. Explainable AI (XAI) Integration

Grad-CAM was used to generate visual heatmaps highlighting suspicious regions, and SHAP provided feature importance scores. The system outputs original X-ray, prediction, and explanation side-by-side.

5. Deployment Methodology

A Streamlit web application was developed where clinicians can upload chest X-rays, receive real-time classification, view prediction confidence, and see interpretability outputs (Grad-CAM and SHAP). The backend runs on GPU (local or cloud) for speed and reliability.

6. Evaluation Metrics

Performance was measured using Accuracy, Precision, Recall, and F1-score (macro and per class), AUROC (macro and per class), confusion matrices, and comparison against baseline models such as ResNet50 and plain DenseNet121.
